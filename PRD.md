# 产品需求文档 (PRD) - 个人 AI 计划助手

**版本：** V1.0  
**定位：** 极简、私密、AI 驱动的养成系计划管理工具。

---

## 1. 产品概述

一款面向开发者个人的 iOS 应用，通过**极简的任务输入**和**本地化存储**保护隐私，引入 **AI 战略分析**与 **Pet 养成系统**，将枯燥的计划管理转化为“升级打怪”的趣味体验。

---

## 2. 核心功能需求

### 2.1 任务管理模块 (Task Management)

- **闪电录入 (Flash Entry)：** 首页常驻单行文本输入框，输入内容后回车即存入“任务池”，默认不强制要求分类或日期。
- **任务属性：**
  - **优先级：** 采用四象限或高/中/低三档。
  - **标签：** 预设【工作】、【学习】、【生活】、【侧线项目】四个维度。
  - **状态机：** 任务包含：待开始 (Backlog)、进行中 (In Progress)、已完成 (Done)、已放弃 (Dropped)。
- **交互逻辑：** 列表项支持左滑“放弃”，右滑“完成”，长按进入“详细编辑（设置截止日期/备注）”。

### 2.2 计划视图 (Planning View)

- **今日看板 (Daily Focus)：**
  - 每日凌晨 4 点重置，自动从池中选出 3-5 件事。
  - 允许用户手动从池中“Pin”任务到今日。
- **日历集成：** 顶部提供周/月切换视图。点击日期可查看：
  1. 该日到期的任务（Deadline）。
  2. 该日实际完成的任务（成就回溯）。
- **倒计时组件：** 对于有 `Due Date` 的任务，在列表右侧显著显示“距截止还剩 X 天”或“已延期”。

### 2.3 AI 助手模块 (AI Pet Assistant)

- **身份设定：** \* **名称：** 用户可自定义。
  - **形象：** 初始形象为一只小狗（如：柯基或柴犬），支持随等级变化的图片/动图资产。
- **AI 核心功能：**
  - **价值分析 (Value Scoring)：** AI 读取未完成任务列表，根据标题语义、优先级、剩余时间，给出 1-100 的“价值分”并生成一句话说明（例如：“这个任务直接影响你的副业进度，优先级极高”）。
  - **行动建议 (Next Move)：** 当用户迷茫时，点击 AI，它会综合分析给出“现在最推荐做的一件事”。

### 2.4 养成系统 (Gamification)

- **XP (经验值) 体系：**
  - **获取：** 完成任务获得 XP。算法公式：$XP = 基础分(10) \times 优先级系数(1/3/5) + AI价值分 \times 0.2$。
  - **惩罚：** 任务延期或“放弃”不扣分，但会触发 AI Pet 的“沮丧”表情。
- **等级进化：**
  - 设置 Level 1-100。每 5-10 级解锁一个新的宠物饰品（如围巾、眼镜）或背景。
- **互动反馈：** 点击宠物会弹出随机语录，语录库根据当前完成率动态选择（如：完成率高时夸奖，低时鼓励）。

## 3. UI/UX 设计要求

| 模块         | 设计要点                                                                           |
| :----------- | :--------------------------------------------------------------------------------- |
| **主题适配** | 强制适配 iOS `System Appearance`，暗黑模式需针对 OLED 屏幕采用纯黑背景以省电。     |
| **视觉风格** | **Neumorphism (新拟物化)** 或 **极简线条风**。减少层级嵌套，通过卡片阴影区分任务。 |
| **反馈动画** | 任务勾选完成时，屏幕出现微小的纸屑喷洒动画；AI 分析时有波纹加载特效。              |

---

## 4. 技术性需求与数据安全

### 4.1 数据架构 (Local Only)

- **持久化方案：** 使用 **SwiftData**。数据严格保存在本地沙盒，不得在未经用户允许的情况下上传至任何第三方云端。
- **隐私策略：** App 不请求联系人、地理位置等敏感权限。

### 4.2 AI 接入方案 (Bridge Mode)

- **中转机制：** iOS 端不存储 API Key。
- **流程：** iOS 端请求 -> 你的 Node.js 中转服务器 (处理 Key 认证与 Prompt 注入) -> 国内大模型 API -> 结果返回给 iOS。
- **离线处理：** 无网络时，屏蔽 AI 分析功能，仅保留基础任务打卡，XP 缓存至下次联网更新。

---

## 5. 核心逻辑流程图 (Logic Flow)

1.  **用户输入任务** -> 进入 Backlog。
2.  **触发 AI 分析** -> Node.js 获取 Task JSON -> 返回 Value Scores -> 更新任务列表排序。
3.  **用户勾选完成** -> 计算 XP -> AI Pet 播放开心动画 -> 更新等级进度条。
4.  **达到升级阈值** -> 弹出 Level Up 界面 -> 更换 Pet 图片 Asset。

## 7.AI请求

默认使用GLM，参考：

```
curl --request POST \
  --url https://open.bigmodel.cn/api/paas/v4/chat/completions \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '
{
  "model": "glm-5",
  "messages": [
    {
      "role": "system",
      "content": "你是一个有用的AI助手。"
    },
    {
      "role": "user",
      "content": "请介绍一下人工智能的发展历程。"
    }
  ],
  "stream": false,
  "temperature": 1
}
'
```

默认不在客户端保存 API Key。

实际接入时请将 `GLM_API_KEY` 写入 Node.js 中转服务的环境变量，例如 `server/.env`，由服务端统一代理请求。
